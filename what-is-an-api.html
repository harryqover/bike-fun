How an Insurance API Works

What is an API?

Think of an API as a digital waiter. Your website (the customer) gives the waiter an order. The waiter takes the order to the kitchen (our systems/database), and brings back the food (the data/result).

What is JSON?

JSON (JavaScript Object Notation) is simply the language the waiter speaks. It's a structured, lightweight text format organized in simple "Key-Value" pairs (like a digital form) that both computers easily understand.

PM Takeaway #1

Notice how we break the flow into two steps. Creating a Quote is harmless and doesn't charge money. It creates an ID that acts as a temporary shopping cart.

PM Takeaway #2

The quote_id is the vital link. The frontend stores it temporarily, and when the user clicks "Buy", we just send that ID to the Policy API endpoint. No need to resend the bike details!

PM Takeaway #3

JSON acts as a strict contract. If the API expects vehicle_type but the frontend sends bike_type, the API will reject it (400 Bad Request error).
